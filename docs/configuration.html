Configuration

Configuration Methods

FORGE supports multiple configuration methods (checked in order):

  1. Command-line flags (highest priority)
  2. Environment variables
  3. Configuration file (~/.forge/config.json)
  4. System keychain (for API keys)
  5. Default values

Environment Variables

Variable Description Required
DEEPSEEK_API_KEY Your DeepSeek API key Yes
GITHUB_TOKEN GitHub personal access token For GitHub features
FORGE_CONFIG_DIR Custom config directory No
FORGE_THEME UI theme (dark/light/glass) No

Configuration Commands

View all config

forge config list

Set a value

forge config set deepseek.api_key sk-xxx

Get a value

forge config get deepseek.api_key

Delete a value

forge config delete deepseek.api_key

Located at ~/.forge/config.json

  }
}

Security & Key Storage

FORGE stores sensitive data securely:

  • API Keys: Stored in system keychain (macOS Keychain, Windows Credential Manager, libsecret on Linux)
  • Config file: Stored in user home directory with 600 permissions
  • Environment variables: Never persisted to disk
  • Tokens: Encrypted at rest
üîí Best Practice: Use environment variables for CI/CD, keychain for local development.
